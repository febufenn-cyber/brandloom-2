# Brandloom Phase 2 — evidence-backed brand memory

## Promise

Every edit can teach Brandloom, but no isolated behaviour becomes a permanent brand rule. Phase 2 preserves evidence, proposes scoped lessons, asks for confirmation when needed, expires temporary facts and records which memories influenced each generation.

## Implemented

- Explicit teach-Brandloom memories
- Candidate memories inferred from before/after edits
- Learning inbox with confirm, reject and pause controls
- Confidence, evidence count, durability, validity dates and scope
- Product, audience, platform and language scoping
- Expiry filtering and reversible memory status
- Memory-aware weekly strategy, writing and regeneration endpoints
- Retrieval provenance linked to generation runs
- Weekly learning reviews
- Repetition reports for hooks, CTAs and content pillars
- Experiments table and API foundation
- Memory export and inferred-memory reset
- RLS for all Phase 2 data

## Memory promotion ladder

1. Raw edit or explicit instruction is preserved.
2. Edit analysis creates one or more candidates.
3. Repeated evidence increases confidence deterministically.
4. Candidates remain outside generation until confirmed or activated.
5. The user may confirm, reject, pause, change scope or add an expiry.
6. Only confirmed and active, non-expired memories enter generation context.

## Memory types

- Voice preference
- Selling style
- Factual rule
- Compliance restriction
- Product lesson
- Audience lesson
- Campaign lesson
- Temporary context
- Repetition warning
- Strategic suggestion

## What remains deliberately deferred

Semantic embeddings and performance-based recommendations remain deferred until the product has enough approved and rejected material to measure whether semantic retrieval improves acceptance. Relational truth, scope, provenance and user control come first.

## Primary Phase 2 measures

- Change in accepted-without-major-rewrite rate
- Median edit distance per post
- Time from generation to approval
- Repeated rejection rate
- Candidate confirmation rate
- Stale-memory incident rate
- Number of generation runs with complete retrieval provenance
